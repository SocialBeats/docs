# Estructura del repositorio

## Customer Agreement

El documento de Customer Agreement se divide en 4 mds diferentes, todos ellos disponibles en la carpeta *legal*. Están disponibles en formato md y pdf. Si se desean visualizar los 4 documentos unidos, está disponible el pdf llamado *customer_agreement_full.pdf*.

## Plan de precios

Los planes de precios están definidos en el documento *pricing.md/pricing.pdf* dentro de la carpeta *legal*. Existen 3 niveles de plan de precios (Free, Pro y Studio).

## Niveles de acabado

Los niveles de acabado se encuentran todos en la carpeta *level_of_finish*. Están disponibles tanto en md como en pdf, y hay tantos como microservicios existen (6) junto con el application.md. Cada pareja ha detallado su nivel de acabado en el documento cuyo nombre coincide con el microservicio desarrollado por ellos. El sexto microservicio (payments_and_suscriptions) tiene un documento propio, y es la única excepción porque fue desarrollado por Ramón Gavira Sánchez y Miguel Encina Martínez, una pareja que se creó sólo para este microservicio pero que de forma separada con otros compañeros desarrollaron otro. Es decir, Ramón  desarrolló con Benjamín el microservicio de usuarios, y Miguel Encina con Daniel Vela el de subida de beats, pero se unieron para crear en colaboración el microservicio de pagos y suscripciones. El séptimo documento es *application.md/application.pdf*, que recoge el nivel de acabado de los requisitos de la aplicación, es decir, el cumplimiento de los requisitos comunes con la nota a la que optamos en la parte grupal.

## Ficha técnica de APIs externas utilizadas y suscripciones óptimas de APIs externas

Ambos documentos se encuentran tanto en md como pdf disponibles en la carpeta *external_apis*. El documento con las fichas técnicas de todas las APIs se llama *datasheet.md/datasheet.pdf*. Cada pareja contribuyó añadiendo las APIs externas utilizadas.
Con respecto al de suscripciones óptimas, se llama *optimal_subscriptions.md/optimal_subscriptions.pdf*. **NOTA:** El microservicio Social no ha utilizado APIs externas.

## Hojas de esfuerzo

El tiempo dedicado al trabajo así como los informes detallados de las tareas realizadas se encuentran disponibles en la carpeta *time_efforts*. Se utilizó Clockify como herramienta de control del tiempo.

El documento de uso de IA se encuentra en formato .md y pdf en la carpeta *ai_use*. Es un documento grupal al que cada pareja ha contribuido.

## Metodología de desarrollo

En la carpeta *work_methodology* podemos encontrar en md y en pdf un documento donde se explica cómo hemos usado Git Flow y qué metodología de commits hemos escogido. Este documento sirve como justificación y prueba de la correcta aplicación de Git Flow durante el desarrollo del proyecto.
